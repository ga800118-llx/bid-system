## 一、颜色系统

所有颜色统一使用 CSS 变量，不允许在业务代码中写死 HEX 值。

### 1.1 语义色

| 变量名 | 色值 | 用途 |
|--------|------|------|
| --color-primary | #1650ff | 主色：按钮、链接、选中态 |
| --color-success | #00B42A | 成功：操作成功、通过状态 |
| --color-warning | #FF7D00 | 警告：提示、异常状态 |
| --color-danger | #F53F3F | 危险：删除、禁用、错误 |
| --color-info | #0EA5E9 | 信息：辅助提示 |

### 1.2 中性色

| 变量名 | 色值 | 用途 |
|--------|------|------|
| --color-text-primary | #1d1d1d | 标题、重要文字 |
| --color-text-regular | #333333 | 正文 |
| --color-text-secondary | #666666 | 辅助说明文字 |
| --color-text-placeholder | #999999 | 占位文字、禁用态 |
| --color-border | #e8e8e8 | 边框、分割线 |
| --color-bg-page | #f0f2f5 | 页面背景 |
| --color-bg-card | #ffffff | 卡片背景 |
| --color-bg-hover | #f5f7fa | 悬停背景 |

### 1.3 暗黑模式

> 换肤时只修改以上 CSS 变量值，不改动业务代码。

## 五、按钮规范

### 5.1 按钮类型

| 类型 | 用途 | 样式 |
|------|------|------|
| primary | 主操作（保存/提交） | 蓝色填充 |
| secondary | 次要操作（取消/返回） | 边框按钮 |
| danger | 危险操作（删除） | 红色 |
| text | 辅助操作（行内编辑） | 文字按钮 |

### 5.2 按钮大小

| 大小 | 适用场景 |
|------|----------|
| large | 表单底部主操作按钮 |
| medium | 工具栏按钮 |
| small | 表格行内操作按钮 |

### 5.3 按钮状态

- 加载中：显示 `loading`，禁止点击
- 禁用：`disabled`，灰化
- 危险操作：使用 `danger` 类型，不使用 `primary`

## 六、表格规范

### 6.1 基础配置

```javascript
// 所有表格统一配置
{
  stripe: true,        // 斑马纹
  size: 'small',      // 紧凑行高
  pagination: true,   // 分页
  resizable: true     // 列宽可拖拽
}
```

### 6.2 列规范

**序号列（必须）：**
- 位置：第一列
- 宽度：固定 70px
- 对齐：居中
- 不参与排序
- 分页场景序号计算：`(page - 1) * size + rowIndex + 1`

**固定列：**
- 首列（序号）：`fixed: 'left'`
- 末列（操作列）：`fixed: 'right'`
- 操作列宽度：固定，不压缩

**可排序列：**
- 时间类（开标时间、创建时间）：支持升序/降序
- 数字类（金额、拦标价）：支持升序/降序
- 操作列：禁止排序

**列宽调整：**
- 所有列默认可拖拽调整宽度（`resizable: true`）
- 首列和末列宽度固定，不受拖拽影响

### 6.3 分页规范

- 位置：右下角
- 默认每页条数：20
- 可选条数：10 / 20 / 50 / 100
- 必须显示 `show-total`，显示「共 N 条」

### 6.4 空状态

- 无数据时显示空图标 + 文字「暂无数据」
- 不显示空白表格骨架

### 6.5 加载状态

- 使用 `loading` 属性
- 显示加载覆盖层，不跳白屏

## 七、表单组件规范

### 7.1 组件分级

| 组件 | 用途 |
|------|------|
| FormInput | 单行文本输入 |
| FormSelect | 下拉选择 |
| FormTextarea | 多行文本 |
| FormDatePicker | 日期选择 |
| FormNumber | 数字输入 |

### 7.2 FormInput 类型与验证规则

| type | 验证规则 | 错误提示 |
|------|----------|----------|
| text | maxLength 限制字符数 | 超过字数时截断并提示 |
| number | 数字 + precision 控制小数位 | 非数字不可输入 |
| phone | /^1[3-9]\d{9}$/ | 手机号格式不正确 |
| email | /^[^\s@]+@[^\s@]+\.[^\s@]+$/ | 邮箱格式不正确 |
| idCard | 18位正则 + 校验位 | 身份证格式不正确 |
| amount | 数字 + 最多2位小数 | 金额格式不正确 |

### 7.3 通用属性

| 属性 | 说明 |
|------|------|
| label | 字段名称 |
| required | 是否必填（显示红色 * 号） |
| placeholder | 占位提示文字 |
| disabled | 禁用状态 |
| maxLength | 最大字符数（text 用） |
| precision | 小数位数（number/amount 用） |

### 7.4 错误提示规范

- 空值未填：请输入XXX / 请选择XXX
- 格式错误：XXX格式不正确
- 超长：XXX不能超过N个字符
- 统一显示在输入框下方，红色 12px 文字

### 7.5 API 示例

```vue
<FormInput
  v-model="form.realName"
  label="真实姓名"
  type="text"
  :max-length="50"
  required
/>
<FormInput
  v-model="form.phone"
  label="手机号"
  type="phone"
  required
/>
<FormInput
  v-model="form.amount"
  label="金额"
  type="amount"
  :precision="2"
/>
```

### 11.1 页面结构

```

  └── Main Content（页面主体）
        └── Card / List / Form
```

- 系统管理页面统一显示面包屑导航
- 格式：首页 / 系统管理 / 当前页面
- 使用 PageHeader 统一组件

## 十二、开发原则

1. 所有颜色使用 CSS 变量，不写死 HEX
2. 表格统一使用标准配置（见 6.1）
3. 表单统一使用 FormInput 系列组件
4. 所有验证规则统一在组件层处理，不在各页面重复写校验逻辑
5. 换暗黑模式只改 CSS 变量文件，不改动业务代码
