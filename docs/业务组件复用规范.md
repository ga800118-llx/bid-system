# 业务组件复用规范

> 更新时间：2026-05-14
> 适用范围：`frontend/src/components` 下跨页面业务组件

## 目标

`frontend/src/design-system` 只沉淀基础 UI 能力。跨页面复用的业务能力必须放在 `frontend/src/components/<domain>`，让页面只负责组合组件、传入数据和调用接口，避免相似业务片段在多个页面复制。

## 分层边界

### design-system

只能放基础 UI 和无业务领域含义的控件：

- 页面头部、顶部栏、操作区
- 筛选条、关键词搜索、高级筛选
- 表格、分页、批量操作
- 输入框、数字、金额、日期、开关、富文本
- 部门、人员、角色、字典、状态等通用选择器
- 标签、状态、头像、空状态、弹窗壳层
- token、图标、校验工具

### business components

放跨页面可复用的业务能力：

- 用户表单、用户列表业务单元格、用户安全展示
- 部门树、部门节点、部门负责人候选
- 消息收件人、抄送人、附件上传
- 待办处理人、待办附件、待办内容结构
- 文件关联、导入结果、删除影响确认
- 其他第二次出现的业务片段

业务组件可以组合 `Ds` 组件，但不得绕开设计系统重新散写一套基础控件样式。

## 当前已沉淀业务组件

| 目录 | 组件 | 使用场景 |
|------|------|----------|
| `frontend/src/components/user` | `UserFormModal.vue` | 新增 / 编辑用户 |
| `frontend/src/components/department` | `DepartmentTreePanel.vue` | 部门管理左侧组织树 |
| `frontend/src/components/message` | `MessageRecipientPicker.vue` | 消息收件人、抄送人、待办处理人选择 |
| `frontend/src/components/message` | `MessageAttachmentUploader.vue` | 消息、待办、文件关联附件上传 |

## 必须复用的场景

用户：

- 新增 / 编辑用户必须使用 `UserFormModal.vue`。
- 用户信息展示必须使用 `DsUserCell`。
- 用户角色展示必须使用 `DsRoleTag`。
- 用户状态展示必须使用 `DsStatusTag`。
- 手机号、邮箱、身份证号必须使用 `DsInput` 对应类型。

部门：

- 部门管理左侧组织树必须使用 `DepartmentTreePanel.vue`。
- 部门选择必须使用 `DsDeptTreeSelect`。
- 部门负责人选择必须使用 `DsUserAvatarSelect`，并传入本部门正常用户候选项。
- 部门树节点状态必须使用 `DsStatusTag`。
- 禁止部门字段显示原始 id。

消息：

- 指定收件人、抄送人必须使用 `MessageRecipientPicker.vue`。
- `MessageRecipientPicker` 必须保持下拉框式部门人员树。
- 部门节点可选中整棵子树，人员节点可单独多选。
- 最终只向业务接口输出用户 ID 数组。
- 消息正文必须使用 `DsRichTextEditor`。
- 消息附件必须使用 `MessageAttachmentUploader.vue`。
- 抄送只做轻量 `CC`，不得扩展为内部邮件系统的回复、转发、草稿箱、发件箱、线程、密送。

待办：

- 待办处理人必须使用 `MessageRecipientPicker.vue`。
- 新增待办必须复用消息中心三段式弹窗结构：基础信息、处理范围、待办内容。
- 待办内容必须使用 `DsRichTextEditor`。
- 待办附件必须使用 `MessageAttachmentUploader.vue`。
- 多处理人待办必须通过处理人明细实现个人状态独立流转，不允许逗号字段或单条状态模拟多人处理。

文件：

- 文件列表必须使用 `DsDataTable / DsPagination / FileTypeIcon / DsStatusTag`。
- 文件关联表单必须使用 `DsModalForm`。
- 文件、消息、待办详情必须使用统一白色详情抽屉样式。

## 什么时候必须抽组件

满足以下任一条件，优先抽业务组件：

- 同一业务表单第二次出现。
- 同一表格列和操作第二次出现。
- 同一弹窗确认逻辑第二次出现。
- 同一筛选组合第二次出现。
- 同一状态展示规则在两个页面出现。
- 单个页面区域超过 150 行且有清晰子区域。
- 业务片段需要同时支持消息、待办、文件、用户、部门等多个页面。

## 组件设计要求

业务组件应遵守：

- 通过 `props` 接收数据。
- 通过 `emits` 通知事件。
- 页面负责接口请求、权限判断、路由跳转和刷新。
- 组件内部不写死接口地址。
- 组件内部不直接读取路由，除非组件本身就是路由级能力。
- 组件状态和错误文案保持可控，方便页面复用。
- 组件必须支持暗黑模式，样式只使用 `--ds-*` token。

## 目录建议

```text
frontend/src/components/
├── user/
├── department/
├── message/
├── file/
├── todo/
└── common/
```

新增领域目录时，应补一个 `README.md`，说明：

- 组件职责。
- 可复用场景。
- 输入输出。
- 依赖的接口或数据结构。
- 不应承担的职责。

## 交付说明模板

每次涉及页面或组件开发，交付说明应包含：

```text
复用组件：
- DsDataTable / DsPagination / DsModalForm
- UserFormModal / MessageRecipientPicker

新增可复用组件：
- frontend/src/components/<domain>/<Component>.vue：用于 ...

暂未抽取：
- xxx 与 yyy 有部分相似，但字段和操作差异较大，暂不抽取；第三次出现时再抽。
```
