# 主题与暗黑模式规范

> 更新时间：2026-05-14
> 适用范围：前端所有页面、基础组件、业务组件

## 当前实现

系统已支持亮色 / 暗色主题切换。

核心文件：

- `frontend/src/utils/theme.js`
- `frontend/src/design-system/tokens/colors.css`
- `frontend/src/design-system/tokens/shadow.css`
- `frontend/src/App.vue`
- `frontend/src/components/Layout.vue`
- `frontend/src/design-system/layout/DsTopBar.vue`

主题状态由 `theme.js` 统一管理：

- 读取本地主题偏好。
- 持久化到 `localStorage`。
- 设置 `document.documentElement.dataset.theme`。
- 设置 Arco 的 `body[arco-theme]`。

业务页面不得自行实现主题状态。

## 主题入口

当前全局切换入口在 `DsTopBar`，由 `Layout.vue` 传入：

- `show-theme-toggle`
- `theme-mode`
- `theme-toggle` 事件

后续不要在单个业务页面单独增加主题切换按钮。

## Token 规则

颜色必须使用 `frontend/src/design-system/tokens/colors.css` 中的 `--ds-*` 变量。

常用 token：

| token | 用途 |
|-------|------|
| `--ds-color-primary` | 主色、选中、链接、主按钮 |
| `--ds-color-success` | 成功状态 |
| `--ds-color-warning` | 警告状态 |
| `--ds-color-danger` | 危险、错误、删除 |
| `--ds-color-info` | 信息提示 |
| `--ds-color-text-primary` | 标题和重要文字 |
| `--ds-color-text-regular` | 正文 |
| `--ds-color-text-secondary` | 辅助文字 |
| `--ds-color-text-placeholder` | 占位、弱提示 |
| `--ds-color-border` | 普通边框 |
| `--ds-color-border-strong` | 强边框 |
| `--ds-color-bg-page` | 页面背景 |
| `--ds-color-bg-layout` | Layout 背景 |
| `--ds-color-bg-card` | 卡片、弹窗、表格背景 |
| `--ds-color-bg-soft` | 次级块背景 |
| `--ds-color-bg-hover` | hover 背景 |
| `--ds-color-bg-selected` | 选中背景 |
| `--ds-color-bg-overlay` | 半透明顶栏、浮层背景 |

阴影必须优先使用：

- `--ds-shadow-card`
- `--ds-shadow-dropdown`
- `--ds-shadow-modal`

## 禁止硬编码

业务页面、业务组件、design-system 新组件中禁止新增：

- `#fff`
- `#f0f2f5`
- `#f6f8fc`
- `#1d2129`
- `#4e5969`
- `#86909c`
- `#e5e6eb`
- 直接用于 UI 背景、文字、边框、阴影的 `rgba(...)`
- `color-mix(..., white)` 或 `color-mix(..., black)` 作为主题基准

允许例外：

- 登录页品牌大图遮罩。
- 图片、图标、业务色预览的固定色。
- 图表兜底色，但应优先读取 CSS token。
- 第三方库要求的不可变色值，但需要在代码旁说明原因。

## `color-mix` 规则

需要混色时，不直接使用 `white` 或 `black`：

```css
/* 推荐 */
background: color-mix(in srgb, var(--ds-color-primary) 8%, var(--ds-color-bg-card));
border-color: color-mix(in srgb, var(--ds-color-primary) 20%, var(--ds-color-border));

/* 禁止 */
background: color-mix(in srgb, var(--ds-color-primary) 8%, white);
```

## 非 CSS 场景

ECharts、Canvas、SVG data URL、动态图片水印等不能直接使用 CSS 变量时，必须读取实际 token 值：

```js
const readThemeVar = (name) => {
  return getComputedStyle(document.documentElement).getPropertyValue(name).trim()
}

const primary = readThemeVar('--ds-color-primary')
```

如果页面中有图表，切换主题后需要重绘或更新图表配置。

## Arco 组件

主题初始化时会设置：

```js
document.body.setAttribute('arco-theme', theme)
```

后续使用 Arco 组件时：

- 优先通过 `Ds` 组件间接使用。
- 必须直接使用 Arco 时，外层样式仍使用 `--ds-*` token。
- 不在业务页面用深选择器强行覆盖 Arco 的暗色主题，除非是修复明确的视觉问题。

## 新增组件要求

新增基础组件或业务组件时：

1. 亮色模式可读。
2. 暗色模式可读。
3. hover、selected、disabled、danger 状态均可读。
4. 弹窗、下拉、popover 不出现白块。
5. 表格、分页、筛选区不出现硬编码浅色。
6. 使用 `rg` 检查硬编码颜色。

推荐检查命令：

```bash
rg -n "#fff|#f0f2f5|#1d2129|#4e5969|#86909c|color-mix\\([^\\n]*(white|black)" frontend/src
```
