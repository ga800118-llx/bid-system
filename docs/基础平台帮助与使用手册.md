# 基础平台帮助与使用手册

> 更新时间：2026-04-21
> 适用范围：当前基础平台封板阶段

这份文档对应系统内 `/help` 帮助页的当前口径，重点说明基础平台已经具备什么、系统管理怎么使用、权限和配置怎么理解，以及新同事接手时应该先看什么。

## 一、平台概览

当前基础平台已经覆盖组织、账号、权限、配置、字典和审计这些通用底座能力，可直接承接后续业务模块。

### 当前已完成模块

| 模块 | 说明 | 状态 |
|---|---|---|
| 部门管理 | 部门树、搜索、新增、编辑、启停、负责人和删除前影响提示 | 已完成 |
| 用户管理 | 分页筛选、新增编辑、启停、调部门、重置密码、多角色分配 | 已完成 |
| 角色权限 | 角色 CRUD、功能权限、数据范围、字段权限三层配置 | 已完成 |
| 操作日志 | 关键写操作审计、权限拒绝审计、登录安全审计和筛选查询 | 已完成 |
| 系统配置 | 统一维护企业名称、系统标题、安全配置、分页和用户水印 | 已完成 |
| 字典管理 | 统一维护字典类型和字典项，服务状态、模块、动作和下拉复用 | 已完成 |

### 当前阶段明确不做

以下内容不属于本轮基础平台封板范围：

- 定时任务
- 文件存储
- 消息通知
- API Key
- 请求限流
- 项目管理、上传、项目删除相关的平台化改造

## 二、系统管理页面

### 页面用途

| 页面 | 主要用途 | 常见操作 |
|---|---|---|
| `/system/department` | 维护组织架构与部门负责人 | 新增部门、编辑部门、启停、查看部门内用户 |
| `/system/user` | 从全局视角维护用户 | 搜索筛选、新增、编辑、重置密码、调部门、分配角色 |
| `/system/role` | 配置角色与权限 | 新增角色、保存功能权限、设置数据范围、配置字段权限 |
| `/system/log` | 查询审计日志与登录安全日志 | 按模块、动作、结果、时间范围和关键词筛选 |
| `/system/config` | 维护系统全局配置 | 搜索、分组筛选、新增配置、编辑配置、启停非内置项 |
| `/system/dict` | 维护字典类型和字典项 | 维护类型、维护字典项、启停、设置默认项与标签色 |

### 推荐使用顺序

1. 先看部门  
   确认组织架构、根部门和负责人设置是否正确。

2. 再看用户  
   新增账号、分配部门、设置角色并检查登录策略。

3. 然后配角色  
   按功能权限、数据范围、字段权限配置角色。

4. 最后收配置与字典  
   统一系统标题、企业名称、水印、分页和下拉字典。

### 管理员首次上线检查清单

- 确认 `enterprise.name` 和 `system.title` 是否已改成正式值
- 确认管理员密码已调整，登录、退出、失败锁定日志能正常记录
- 确认根部门、部门层级和负责人设置正确，未分配用户已清理
- 确认关键角色的功能权限、数据范围和字段权限配置符合预期
- 确认分页、水印、密码策略、侧边栏默认状态等基础配置符合使用场景
- 确认通用状态、日志模块、日志动作、配置分组等内置字典正常可用
- 确认新增、编辑、启停、登录失败、权限拒绝等事件都能在操作日志中查询到

## 三、权限说明

### 权限模型

- 功能权限  
  决定是否能进入页面、看到菜单、使用新增编辑删除等操作。

- 数据范围  
  决定用户可查看哪些部门或用户数据，例如全部、本部门、本人。

- 字段权限  
  决定某个字段是否可读、是否可写，当前已在用户管理和操作日志落地。

### 字段权限当前覆盖范围

| 模块 | 受控字段 | 作用 |
|---|---|---|
| `system_user` | 账号、姓名、部门、角色、状态、登录安全 | 控制列表列显示、编辑表单可写性和接口返回字段 |
| `system_log` | 操作人、IP、请求信息、操作内容 | 控制日志列表列显示和接口字段返回 |

说明：当前字段权限第一版只覆盖用户管理和操作日志，这是当前阶段有意收住的范围。

### 当前权限口径

- 普通用户的菜单、按钮和接口访问，以角色配置为准
- 系统管理员仍保留兼容兜底，默认拥有全部权限
- 数据范围决定“能看到哪些数据”，字段权限决定“能看到哪些列、能改哪些字段”
- 权限拒绝、登录成功、登录失败、账号锁定、退出登录都会写入操作日志

## 四、系统配置与字典

### 系统配置适合放什么

| 配置项类型 | 典型示例 | 说明 |
|---|---|---|
| 品牌信息 | `enterprise.name`、`system.title` | 统一管理企业名称和系统名称 |
| 安全策略 | 密码最小长度、强密码、登录失败次数、会话超时 | 用于登录和密码规则统一控制 |
| 界面偏好 | 默认分页、分页选项、侧边栏默认折叠 | 用于基础交互体验统一 |
| 全局水印 | 开关、模板、透明度、字号、旋转角度 | 用于后台页面水印审计和信息保护 |

### 用户水印配置

| 配置键 | 含义 | 常见值 |
|---|---|---|
| `ui.watermark.enabled` | 控制后台页面是否显示水印 | `true / false` |
| `ui.watermark.text_template` | 定义水印内容模板 | `{realName} {username} {dateTime}` |
| `ui.watermark.opacity` | 控制水印透明度 | `0.12` |
| `ui.watermark.font_size` | 控制水印字号 | `16` |
| `ui.watermark.rotate` | 控制水印旋转角度 | `-24` |

### 字典管理怎么用

- 字典类型用于定义一组固定选项，例如状态、模块、动作、分组
- 字典项用于定义具体选项，例如启用、禁用、基础设置、安全设置
- 适合放入字典的内容通常是下拉选项、标签状态、筛选条件和未来可能扩展的枚举值
- 后续开发业务模块时，建议先判断是否需要字典，再开发页面和接口

## 六、常见问题

### 系统管理员是不是必须在角色页里把权限全部勾满？

当前不是。系统管理员仍保留兼容兜底，即使角色页未完整勾选，默认仍拥有全部权限。

### 为什么字段权限现在只覆盖用户管理和操作日志？

这是当前阶段有意收住的范围。先覆盖最敏感、最有价值的模块，后续业务模块再按需要逐步接入。

### 新增业务模块时，应该先做什么？

建议先判断这个模块是否需要系统配置、字典、权限点、数据范围和字段权限，再开始页面和接口开发。

### 哪些内容更适合放到字典管理里？

下拉选项、状态标签、筛选条件、未来可能扩展的枚举值，通常都适合放进字典。

### 为什么这里不再详细介绍上传和 Prompt？

当前基础平台已经接近封板，这份帮助文档以基础平台底座为主；项目、上传和 Prompt 相关能力后续更适合单独整理业务帮助文档。
