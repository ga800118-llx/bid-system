# 仓库文件整理清单

> 更新时间：2026-04-21
> 目的：区分源码资产、本地运行产物和历史备份，减少后续开发干扰

## 整理原则

当前工作树较脏，不能简单按“看起来多余”就删除。

本项目整理时建议按三类处理：

1. **源码与文档**
   - 必须保留并继续维护

2. **本地运行产物**
   - 不作为源码理解
   - 优先通过 `.gitignore` 隔离
   - 非必要不直接删除

3. **历史备份/临时输出**
   - 先确认是否仍有用途
   - 再决定归档或删除

## 一、应保留并持续维护

### 源码

- `backend/src/`
- `frontend/src/`
- `backend/local-schema.sql`
- `frontend/package.json`
- `frontend/vite.config.js`
- `backend/pom.xml`

### 文档

- `README.md`
- `DEVELOPMENT.md`
- `docs/`

### 本地运行必要依赖

- `.tools/`

说明：

`.tools/` 虽然不属于业务源码，但当前本地开发高度依赖它，不应随意删除。

## 二、应视为本地运行产物

这些内容默认不应参与日常源码协作理解：

- `backend/.local-db/`
- `backend/*.log`
- `backend/*.gz`
- `backend/*.pdf`
- `backend/*.xlsx`
- `uploads/`
- 根目录 `backend.log`
- `.uuremote_*`

当前处理建议：

- 保留本地调试数据
- 用 `.gitignore` 隔离
- 不再把这类文件当成“项目代码的一部分”

## 三、建议后续确认后再处理

### SQL 备份

- `douge_bid_backup.sql`
- `douge_bid_backup_utf8.sql`
- `douge_bid_backup_utf8mb4.sql`
- `backend/extract_logs.sql`

建议：

- 先确认这些是否仍需要作为迁移参考
- 如果只是个人备份，建议移出仓库目录或归档到单独备份目录

### 历史业务输出目录

- `C:/`

说明：

这个目录明显带有早期本地脚本/路径兼容痕迹，不建议直接删除。

建议后续确认：

- 是否仍被历史 PDF 提取链路依赖
- 是否只是早期 Windows 路径迁移残留

确认无用后再归档或删除。

### 后端历史样例文件

- `backend/*.pdf`
- `backend/*.xlsx`

说明：

它们更像历史上传或处理样例，而不是后端源码组成部分。

建议：

- 如果还要用于回归测试，可集中迁移到单独样例目录
- 如果只是遗留输出，可后续统一归档

## 四、已经完成的低风险整理

本轮已处理：

- 删除空的 `.uuremote_*`
- 删除 `commit_msg.txt`
- 删除空的 `backend/mvn_err.txt`
- 删除根目录 `backend.log`
- 补充 `.gitignore`，隔离运行产物和本地备份

## 五、后续整理建议

推荐下一步按下面顺序继续：

1. 确认 `C:/` 是否仍有历史脚本依赖
2. 盘点根目录和 `backend/` 下的 SQL 备份文件
3. 决定历史 PDF / Excel 样例是否迁到单独 `samples/` 或 `archives/`
4. 若后续进入多人协作，进一步压缩工作树中的本地产物

## 六、原则总结

后续整理仓库时，优先遵守这三条：

1. 不误删本地数据库和调试数据
2. 不把运行产物继续当成源码资产
3. 先隔离，再归档，再删除
