# 角色权限页样板说明

## 目的

`/system/role` 当前承担基础平台复杂配置页样板角色。

这份说明用于明确：

- 复杂配置页在当前项目中的标准信息层级
- 哪些 design-system 组件和结构已在该页验证通过
- 后续类似页面应复用哪些模式

## 当前样板范围

角色权限页当前已完成并确认的样板范围：

- 页面头部
  - `DsPageHeader`
  - `DsHeaderActions`
- 摘要卡片
  - `DsStatsCard`
- 筛选查询
  - `DsFilterBar`
  - `DsKeywordSearch`
  - `DsStatusSelect`
- 主体双栏结构
  - 左侧角色列表卡片
  - 右侧权限配置卡片
- 列表区
  - `DsDataTable`
  - `DsActionBar`
  - `DsIconButton`
  - `DsStatusTag`
- 弹窗表单
  - `DsModalForm`
  - `DsFormSection`
  - `DsFormGrid`
  - `DsFormActions`
  - `DsReadonlyField`
  - `DsInput`
  - `DsTextarea`
  - `DsStatusSelect`

## 标准复杂配置页结构

后续复杂配置页优先参考以下层级：

1. 页面标题层
   - 面包屑
   - 页面标题
   - 页面说明
   - 页面级主操作 / 工具操作
2. 摘要层
   - 关键数量、状态、受保护对象等摘要卡片
3. 筛选层
   - 关键词
   - 状态
   - 查询 / 重置
4. 主体层
   - 左侧对象列表
   - 右侧配置详情 / 配置面板

## 已确认的通用模式

### 左侧对象列表

- 左栏用于承载“当前要配置哪个对象”
- 桌面端宽度建议 250px 到 280px，左栏固定宽度，右栏自适应
- 顶部使用标题 + 说明 + 刷新工具
- 对象列表使用紧凑选择列表，角色名称 14px / 500，账号数量和编码 12px / 400 弱文本
- 内置状态、分配人数等摘要信息使用小标签展示
- 当前选中行使用浅蓝背景 + 左侧高亮条，不额外发明独立列表样式

### 右侧配置面板

- 右栏用于承载“当前对象的配置内容”
- 顶部展示：
  - 当前对象名称
  - 对象编码
  - 状态
  - 保存说明
  - 保存 / 刷新操作
- 未选择对象时，使用统一空状态，不直接展示空白配置区

### 配置维度分离

- 不同配置维度通过 Tab 分离
- 每个 Tab 顶部提供简短说明文案
- 每个 Tab 保持自己的信息分组边界，不直接堆成大表

### 功能权限区

- 按模块分组展示
- 每个模块使用白色卡片、圆角、弱阴影
- 模块标题使用 14px 到 16px / 600
- 模块说明使用 12px 到 14px 弱文本
- 模块间距保持 20px 到 24px
- 每组保留全选 / 清空，放在模块卡片顶部右侧
- 勾选项横向排列，控件高度保持 32px 到 36px，文字不换行，间距保持 16px 到 20px
- 勾选项文字过长时单行省略，必要时使用轻量白色信息浮层展示完整内容
- 已选项应有浅蓝背景或蓝色文字高亮，不做复杂树状嵌套

### 数据范围区

- 使用轻量表格承载模块与范围规则
- 每行只表达一个模块的一项核心配置

### 字段权限区

- 按模块分组
- 每个字段明确展示：
  - 字段名称
  - 可读
  - 可写
- 可写依赖可读时，界面上要能直接看清关系

## 后续页面应直接复用的内容

以下场景优先复用角色权限页样板：

- 字典规则配置页
- 模板规则配置页
- 审批流节点配置页
- 任何“左侧对象列表 + 右侧配置面板”的后台页面

优先复用的是：

- 双栏布局
- 左栏对象选择模式
- 右栏配置头部模式
- Tab 分维度配置模式
- 新增 / 编辑弹窗表单模式

不要求完全照搬的是：

- Tab 数量
- 左栏列内容
- 配置项具体字段
- 摘要卡片文案

## 当前样板价值

角色权限页当前的价值有三点：

- 它是基础平台第一张复杂配置页样板
- 它验证了“列表页样板”之外的另一类后台页面结构
- 它为后续复杂配置页面提供了统一的布局和交互基线

## 后续维护规则

- 如果角色权限页后续继续调整结构或视觉，必须评估是否影响复杂配置页样板地位
- 如果改动上升为通用规范，必须同步更新：
  - `docs/UI组件库规范.md`
  - `AGENTS.md`
- 如果改动只是角色权限页业务个性，不要误记为所有复杂配置页都必须遵守
