# 权限管理系统设计

> 设计时间：2026-04-09
> 定位：通用权限管理底座，支持多业务系统复用

---

## 核心理念

把权限体系拆成两层：

业务系统（如招投标）
  注册自己的 Feature（菜单/按钮/字段权限）
               |
               v
     通用权限服务（可独立部署）
     Feature层：能不能操作
     DataScope层：能看哪些数据

用户 - 属于组织架构（树形部门） - 拥有多个角色
                                            |
                                Feature权限（能做哪些操作）
                                DataScope（能看哪些数据范围）
                                Field权限（能看到哪些字段）

## 一、组织架构

### 树形结构

- 支持多级：公司 > 部门 > 科室 > 小组
- 也兼容两级：公司 > 门店
- 用户属于一个叶子部门

### 表结构

`sql
org_department (

  username      VARCHAR(50) UNIQUE,
  password_hash VARCHAR(255),
  dept_id       BIGINT,

  created_at    DATETIME
)
`

## 二、权限类型

### 1. Feature 权限（功能操作）

权限码格式：模块:操作

示例：
- bid_project:read - 查看项目
- bid_project:write - 新增/编辑项目
- bid_project:delete - 删除项目
- bid_project:export - 导出
- finance:read - 财务模块读
- finance:pay - 付款操作

### 2. Field 权限（字段级）

示例：
- bid_project:field:ceilingPrice:read - 能看拦标价
- bid_project:field:bidBond:write - 能编辑保证金

### 3. Data Scope（数据可见范围）

每条规则是一个范围表达式，运行时由业务系统根据用户上下文计算。

| 规则 | 含义 | SQL 转化 |
|------|------|----------|
| OWN | 只看自己创建的 | WHERE user_id = :currentUserId |
| DEPT | 本部门 | WHERE dept_id = :userDeptId |
| DEPT_SUB | 本部门及下级部门 | WHERE dept_id IN (本部门+所有子部门) |
| DEPT_UP | 本部门及上级部门 | WHERE dept_id IN (本部门+所有父部门) |
| ALL | 全部 | 不过滤 |

## 三、权限运算规则

### 多角色 - 取并集

用户权限 = 角色A的Feature 并 角色B的Feature
用户数据范围 = 角色A的Scope 并 角色B的Scope（满足任一即可访问）
用户字段权限 = 角色A的Field 并 角色B的Field

### 实时生效

- JWT 只存 userId，不存权限
- 每次请求从 DB 加载权限
- 加 Redis 缓存：变更权限时删缓存，下次请求重新加载

权限变更 - 删除 Redis 缓存 - 下次请求重新加载 - 实时生效

## 四、数据库表结构

`sql
-- 角色表
org_role (
  id          BIGINT PRIMARY KEY,
  code        VARCHAR(50) UNIQUE COMMENT '角色编码',
  name        VARCHAR(100) COMMENT '角色名称',
  description VARCHAR(255),
  created_at  DATETIME
)

-- Feature权限（能做哪些操作）
org_role_feature (
  role_id      BIGINT,
  feature_code VARCHAR(100),
  PRIMARY KEY (role_id, feature_code)
)

-- 数据范围（能看哪些数据）
org_role_data_scope (
  role_id      BIGINT,
  module_code  VARCHAR(50) COMMENT '模块编码',
  scope_rule   VARCHAR(20) COMMENT 'OWN/DEPT/DEPT_SUB/DEPT_UP/ALL',
  PRIMARY KEY (role_id, module_code)
)

-- 字段权限（能看到哪些字段）
org_role_field (
  role_id     BIGINT,
  module_code VARCHAR(50),
  field_code  VARCHAR(50),
  can_read    TINYINT DEFAULT 1,
  can_write   TINYINT DEFAULT 0,
  PRIMARY KEY (role_id, module_code, field_code)
)

-- 用户-角色（多对多）
org_user_role (
  user_id BIGINT,
  role_id BIGINT,
  PRIMARY KEY (user_id, role_id)
)
`

## 五、API 接口设计

认证：
  POST   /auth/login              登录
  GET    /auth/userinfo           当前用户信息+所有角色+所有权限码

角色管理：
  GET    /roles                   角色列表
  POST   /roles                   创建角色
  PUT    /roles/{id}/features     分配功能权限
  PUT    /roles/{id}/data-scope   分配数据范围
  PUT    /roles/{id}/fields       分配字段权限

用户管理：
  GET    /users                   用户列表
  POST   /users                   创建用户
  PUT    /users/{id}/roles        给用户分配角色（支持多角色）

组织架构：
  GET    /departments/tree         组织架构树
  POST   /departments              新增部门
  PUT    /departments/{id}         编辑部门
  DELETE /departments/{id}         删除部门

## 六、权限检查流程

`
请求进入
    |
从 JWT 拿 userId
    |
从 Redis 加载用户权限（无缓存则查DB）
    |
  1. Feature 检查：@RequirePermission("bid:write")
     无权限 - 返回 403
     |
  2. DataScope 注入：自动拼接 SQL WHERE 条件
     查询时自动带上范围过滤
     |
  3. Field 检查：返回前端前过滤不可见字段
     Response 过滤掉 can_read=0 的字段
`

### Spring Boot 注解示例

`java
@RequirePermission("bid_project:write")
public Result saveProject(ProjectDTO dto) { ... }
`

`java
@DataScope(module = "bid_project")
public List<Project> list(ProjectQuery q) { ... }
`

## 七、前端菜单/按钮权限

- 登录后一次性加载用户所有权限码，存 Vuex/Pinia
- 菜单：根据权限码动态显示/隐藏
- 按钮：使用 v-permission 指令控制
- 路由守卫：权限不足则重定向或提示

---

## 八、实现顺序

### 第一步：基础骨架
- 组织架构 CRUD（部门树）
- 用户管理 + 多角色分配
- 角色管理 + 三种权限分配

### 第二步：权限检查接入
- Spring Boot 拦截器 + 注解
- 前端菜单/按钮根据权限动态显示

### 第三步：数据范围接入
- DataScope SQL 自动注入
- 字段级别动态过滤

## 九、与现有招投标系统的集成

现有系统的用户/角色迁移：

1. 保留现有的 sys_user 表，通过 org_user 表关联或直接迁移
2. 现有 admin 角色 - 迁移为拥有所有 Feature + ALL DataScope 的系统管理员
3. 现有 user 角色 - 按具体功能分配对应权限码

Feature 码建议（招投标模块）：

| 权限码 | 含义 |
|--------|------|
| bid_project:read | 查看项目列表和详情 |
| bid_project:write | 上传文件/新增项目 |
| bid_project:delete | 删除项目 |
| bid_project:export | 导出项目数据 |
| bid_prompt:write | 修改 Prompt 配置 |
| bid_user:manage | 用户管理（admin 专属） |
