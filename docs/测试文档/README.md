# 测试文档

## 1. 测试范围

本系统为核心业务系统，主要测试范围包括：
- 用户认证模块（注册 / 登录 / Token 校验）
- 项目管理模块（上传 / 列表 / 详情 / 下载 / 删除）
- AI 信息提取准确性

---

## 2. 测试用例

### 2.1 用户模块

#### TC-001：正常注册

| 项目 | 内容 |
|------|------|
| 前置条件 | 系统无该邮箱账号 |
| 输入 | username= test@example.com，password= 123456 |
| 预期结果 | 注册成功，提示"注册成功"，跳转登录页 |
| 优先级 | P0 |

#### TC-002：重复注册

| 项目 | 内容 |
|------|------|
| 前置条件 | 账号 test@example.com 已存在 |
| 输入 | username= test@example.com，password= 123456 |
| 预期结果 | 提示"该邮箱已注册"，注册失败 |
| 优先级 | P0 |

#### TC-003：正常登录

| 项目 | 内容 |
|------|------|
| 前置条件 | 存在账号 admin@bid.com，密码 admin123 |
| 输入 | username= admin@bid.com，password= admin123 |
| 预期结果 | 登录成功，Token 存入 localStorage，跳转首页 |
| 优先级 | P0 |

#### TC-004：密码错误登录

| 项目 | 内容 |
|------|------|
| 前置条件 | 存在账号 admin@bid.com |
| 输入 | username= admin@bid.com，password= wrongpass |
| 预期结果 | 提示"密码错误"，登录失败 |
| 优先级 | P0 |

#### TC-005：未登录访问受保护接口

| 项目 | 内容 |
|------|------|
| 前置条件 | 未登录或 Token 已过期 |
| 输入 | GET /api/project/list（无 Header） |

### 2.2 项目模块

### 2.3 AI 提取模块

#### TC-020：PDF 文本提取完整性

| 项目 | 内容 |
|------|------|
| 测试方法 | 上传同一份 PDF，用 AI 提取，对比人工标注结果 |
| 评估维度 | 字段覆盖率（14个字段有多少能提取出来）、准确率 |
| 验收标准 | 核心字段（项目名称、编号、开标时间、拦标价）覆盖率 >= 95% |
| 优先级 | P0 |

#### TC-021：关键金额字段准确性

| 项目 | 内容 |
|------|------|
| 测试方法 | 对比提取的拦标价与原文件中的数值 |
| 验收标准 | 拦标价数值完全匹配，误差为 0 |
| 优先级 | P0 |

#### TC-022：日期时间解析准确性

| 项目 | 内容 |
|------|------|
| 测试方法 | 对比提取的开标时间与原文件 |
| 验收标准 | 日期格式统一为 YYYY-MM-DDTHH:mm:ss |
| 优先级 | P1 |

#### TC-023：主观分与客观分区分

| 项目 | 内容 |
|------|------|
| 测试方法 | 检查提取结果中 subjectiveScoreDetails 是否只含主观评分项 |
| 验收标准 | 主观分字段不含业绩数量、资质证书数量等客观评分项 |
| 优先级 | P1 |
