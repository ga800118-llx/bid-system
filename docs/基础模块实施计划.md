## 整体架构

```text
┌──────────────────────────────────────────────┐
│              业务系统层（招投标等）             │
├──────────────────────────────────────────────┤
│              通用组件层                       │
│  文件存储 | 字典管理 | 消息通知 | 定时任务      │
├──────────────────────────────────────────────┤
│              基础平台层                       │
│  部门管理 | 用户管理 | 角色权限 | 操作日志 | 系统配置 │
└──────────────────────────────────────────────┘
```

当前目标：先把基础平台层打稳，不做项目管理、上传、项目删除相关功能扩展。

## 与现有招投标系统的关系

现有系统逐步依赖基础平台层：

```text
部门管理 -> 用户管理 -> 角色权限 -> 操作日志
```

迁移步骤：

1. 先搭建并稳定部门、用户、角色基础能力
2. 将现有 `admin/user` 角色映射到新权限体系
3. 把系统配置、企业名称、系统标题、菜单权限等从硬编码迁出
4. 操作日志记录平台层和关键业务操作
5. 后续再将项目、上传等业务模块逐步接入权限、审计、配置和字典底座
