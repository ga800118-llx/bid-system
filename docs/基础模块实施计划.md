## 整体架构

`
┌──────────────────────────────────────────────┐
│              业务系统层（招投标等）              │

## 实施顺序

### 第一阶段：平台基础骨架（P0）

#### 1. 部门管理
- 树形部门 CRUD
- 部门启用/禁用
- 部门排序
- 禁止删除有用户或子部门的部门

#### 2. 用户管理
- 用户列表 + 分页 + 搜索
- 新增/编辑用户（关联部门、设置角色）
- 禁用/启用用户
- 重置密码（admin 操作）
- 修改个人信息
- 修改密码（本人）

#### 3. 操作日志
- 记录：userId + 操作类型 + 操作内容 + 时间 + IP
- 操作类型：登录/登出/新增/编辑/删除/导出
- 查询：按用户/时间/操作类型筛选
- 保留时间：建议 90 天

#### 4. 角色权限管理
- 角色 CRUD
- Feature 权限分配（功能操作）
- DataScope 分配（数据范围）
- Field 权限分配（字段级）
- 用户-角色分配

### 第二阶段：权限接入（P0）

#### 5. 权限接入业务系统
- Spring Boot 拦截器 + @RequirePermission 注解
- DataScope SQL 自动注入
- 前端菜单/按钮权限动态化

### 第三阶段：通用组件（P1）

#### 6. 系统配置
- key-value 配置表
- 后台增删改查
- 业务系统可读取

#### 7. 字典管理
- 字典类型 + 字典项
- 后台管理
- 前端下拉组件

#### 8. 定时任务
- 可配置 Cron 表达式
- 启用/禁用
- 执行日志

## 与现有招投标系统的关系

现有系统依赖基础平台层的模块：

`
部门管理 ── 用户管理 ── 角色权限 ── 操作日志
    │
    └── 现有 sys_user/sys_role 迁移到新体系
`

迁移步骤：
1. 先搭建新基础平台（部门+用户+角色）
2. 将现有 admin/user 角色映射到新权限体系
3. 操作日志记录关键业务操作
