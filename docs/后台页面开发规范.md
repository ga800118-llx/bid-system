# 后台页面开发规范

> 更新时间：2026-05-14
> 适用范围：`frontend/src/views` 下所有登录后后台页面

## 目标

后续新增或修改后台页面时，必须继承当前基础平台已经形成的页面样板、组件体系和交互节奏。页面开发的目标不是“能显示”，而是和现有后台页面保持同一套结构、密度、操作方式和主题能力。

## 开发前检查

动任何页面前先做复用检查：

1. 识别页面类型。
2. 找到对应样板页。
3. 检查 `frontend/src/design-system` 是否已有基础组件。
4. 检查 `frontend/src/components/<domain>` 是否已有业务组件。
5. 如果现有组件能力不足，优先扩展组件或抽业务组件。
6. 最后再写业务页面组合逻辑。

交付时必须说明：

- 本次复用了哪些组件。
- 本次新增了哪些可复用组件。
- 哪些重复点暂未抽取，以及原因。

## 页面类型与样板

| 页面类型 | 样板页 | 适用场景 | 必须对齐内容 |
|----------|--------|----------|--------------|
| 标准列表管理页 | `frontend/src/views/UserManage.vue` | 用户、文件、消息、待办、配置列表等 | 页面宽度、筛选区、统计区、表格、分页、弹窗表单、行内操作 |
| 左树右表管理页 | `frontend/src/views/Department.vue` | 组织、分类、目录、区域、左侧对象加右侧列表 | 左侧树卡片、选中态、节点操作、右侧列表、批量操作 |
| 复杂权限 / 配置页 | `frontend/src/views/RoleManage.vue` | 权限配置、规则配置、策略配置、字段配置 | 左侧对象列表、右侧配置面板、Tab、模块卡片、保存区 |
| 日志审计页 | `frontend/src/views/OperationLog.vue` | 操作日志、登录日志、任务记录、接口调用记录 | 筛选组合、字典选择、时间范围、详情弹窗、表格节奏 |
| 配置类页面 | `frontend/src/views/SystemConfig.vue`、`frontend/src/views/DictManage.vue` | 参数配置、字典、状态、基础数据维护 | 配置列表、状态标签、编辑弹窗、分组、值类型、状态选择 |

只复用几个 `Ds` 基础组件，但重新拼一套页面结构，不符合规范。

## 页面外壳

- 后台页面必须挂在主 `Layout.vue` 子路由下。
- 路由必须配置 `meta.title`。
- 页面顶部由 `Layout.vue` 和 `DsTopBar` 统一管理。
- 业务页面不得自行实现顶部栏、用户信息、退出按钮、主题切换、多标签页。
- 页面头部优先使用 `DsPageHeader + DsHeaderActions`。
- 页面主体节奏优先对齐对应样板页，不为单页自定义大面积布局风格。

## 列表页

列表页优先使用：

- `DsPageHeader`
- `DsHeaderActions`
- `DsStatsCard` / `DsSecuritySummaryCard`
- `DsFilterBar`
- `DsAdvancedFilterPanel`
- `DsKeywordSearch`
- `DsDataTable`
- `DsActionBar`
- `DsBatchActions`
- `DsPagination`

表格规则：

- 表格壳层必须使用 `DsDataTable`。
- 分页必须使用 `DsPagination`。
- 用户信息使用 `DsUserCell`。
- 角色使用 `DsRoleTag`。
- 状态使用 `DsStatusTag`。
- 安全状态使用 `DsSecurityStatus`。
- 行内操作统一外露主操作，次级或危险动作放入 `更多`。
- 不允许业务页覆盖整张表格的表头、行高、hover、checkbox、分页视觉。

## 表单与弹窗

弹窗、抽屉、内页表单优先使用：

- `DsModalForm`
- `DsFormSection`
- `DsFormGrid`
- `DsFormActions`
- `DsReadonlyField`
- `DsInput`
- `DsNumberInput`
- `DsAmountInput`
- `DsTextarea`
- `DsSwitch`
- `DsDatePicker`
- `DsDateRangePicker`

选择器必须复用：

- 部门：`DsDeptTreeSelect`
- 人员：`DsUserAvatarSelect`
- 角色：`DsRoleMultiSelect`
- 字典：`DsDictSelect`
- 状态：`DsStatusSelect`
- 日期范围：`DsDateRangePicker`

字段规则：

- 手机号：`DsInput type="phone"`
- 邮箱：`DsInput type="email"`
- 身份证号：`DsInput type="idCard"`
- 数字、排序：`DsNumberInput`
- 金额：`DsAmountInput`
- 富文本：`DsRichTextEditor`

业务页面禁止直接拼 `a-modal + a-form` 实现新增 / 编辑表单。

## 详情与抽屉

- 文件、消息、待办详情使用统一白色详情抽屉样式。
- 操作日志详情使用 `DsModalForm` 信息弹窗。
- 业务详情提示、状态详情提示使用白色 Popover 类组件。
- 表格中禁止使用黑底大块 Tooltip 展示业务详情。

## 主题适配

所有后台页面必须支持亮色和暗色主题：

- 颜色使用 `frontend/src/design-system/tokens` 中的 `--ds-*` 变量。
- 不允许业务页面硬编码白底、浅灰、文字色、边框色、阴影色。
- `box-shadow` 优先使用 `--ds-shadow-card / --ds-shadow-dropdown / --ds-shadow-modal`。
- `color-mix` 的基准色应使用 `var(--ds-color-bg-card)`、`var(--ds-color-border)` 等 token，不直接混入 `white`。
- ECharts、Canvas、SVG data URL 等非 CSS 场景必须通过 `getComputedStyle(document.documentElement)` 读取 token。

## 禁止事项

- 禁止新增页面绕开样板页单独设计新风格。
- 禁止在业务页面重复实现部门、人员、角色、状态、日期范围选择器。
- 禁止在业务页面临时拼接统计卡片、筛选条、表格头部、批量操作。
- 禁止在业务页面直接散写 `a-pagination`。
- 禁止把每页条数、总数、跳转页拆成不同布局。
- 禁止为单页需求直接改全局样式绕开 `design-system`。
- 禁止新增硬编码颜色破坏暗黑模式。

## 提交前检查

提交前至少检查：

- 路由是否配置 `meta.title`。
- 页面是否选择并对齐了样板页。
- 是否复用了 `Ds` 基础组件和已有业务组件。
- 是否没有新增硬编码颜色。
- 是否在亮色和暗色下都能阅读和操作。
- `npm run build` 是否通过。
