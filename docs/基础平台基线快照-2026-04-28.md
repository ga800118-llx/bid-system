## 五、当前已知边界

这些不是“遗漏”，而是当前基线的明确边界：

- 字段权限只覆盖 `system_user`、`system_log`
- 文件中心当前仍是本地存储，不含 OSS / MinIO 抽象
- 消息中心当前没有做复杂归档、业务聚合、批量处置
- 待办中心当前没有做复杂催办、审批流、批量操作
- 定时任务、API Key、限流、外部消息通道不在当前基线范围

## 六、后续打磨时必须重点保护的能力

后续做 UI 或交互重构时，至少不要改丢以下闭环：

- 部门 / 用户 / 角色 / 日志 / 配置 / 字典 6 个系统页可正常访问
- 权限体系真实生效：
  - Feature
  - DataScope
  - Field
- 系统配置可真实驱动：
  - 企业名称
  - 系统标题
  - 用户水印
- 文件中心主链可用：
  - 上传
  - 下载
  - 预览
  - 关联
- 消息和待办可联动：
  - 创建待办
  - 生成消息
  - 从消息跳到待办
- 导入导出主链不被破坏
