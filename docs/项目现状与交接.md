> 更新时间：2026-04-26
> 项目路径：`/Users/myfile/bid-system`

## 一句话判断

这个项目当前已经具备一套可复用的后台基础平台底座，适合继续承接大多数业务系统常见的管理后台模块。

当前底座重点包括：

- 组织架构
- 用户与多角色
- 角色权限
- 数据范围
- 字段权限
- 操作日志
- 系统配置
- 字典管理
- 文件中心
- 站内消息中心
- 待办中心
- 导入导出基础能力（系统页导出第一版已齐，字典管理/系统配置/用户/部门导入第一版已完成）

它不是一个“所有通用平台能力都做完”的平台，但已经覆盖了大多数业务系统最常见、最先需要的基础模块。

## 当前阶段明确不做

这些不是当前基础平台封板范围：

- 定时任务
- 外部消息通道（邮件/短信/微信）
- API Key 管理
- 请求限流
- 项目管理权限平台化改造
- 上传 / 项目删除相关功能扩展

后续如果要做，应作为“新模块或新专题”单独规划，不再算基础平台收口。

## 当前系统口径

### 权限

- 当前主权限模型为：
  - Feature
  - DataScope
  - Field

- 当前系统管理员仍保留兼容兜底：
  - `users.role = admin` 时视为超级管理员
  - 不依赖角色页是否完整勾选

- 普通角色权限以数据库角色配置为准

### 字段权限

当前字段权限第一版只覆盖：

- `system_user`
- `system_log`

这不是缺失，而是当前阶段有意收住范围。后续新增业务模块时，再按实际价值逐步接入。

### 字典

字典管理当前主要服务：

- 通用状态
- 用户角色类型
- 配置分组
- 配置值类型
- 操作日志模块
- 操作日志动作
- 标签颜色

后续开发业务模块时，应先判断是否需要先定义字典，再开发页面和功能。
