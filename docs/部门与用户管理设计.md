## 一、数据库表结构

### 1.1 org_department（部门表）

```sql
CREATE TABLE org_department (
  id          BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT ''部门ID'',
  name        VARCHAR(100)  NOT NULL COMMENT ''部门名称'',
  parent_id   BIGINT COMMENT ''父部门ID，NULL=根节点'',
  sort_order  INT DEFAULT 0 COMMENT ''同级排序，数字越小越靠前'',
  status      TINYINT DEFAULT 1 COMMENT ''1=正常 0=禁用'',
  created_at  DATETIME,
  updated_at  DATETIME,
  INDEX idx_parent (parent_id)
) COMMENT=''部门表'';
```

### 1.2 org_user（用户表）

```sql
CREATE TABLE org_user (
  id              BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT ''用户ID'',
  username        VARCHAR(50)  NOT NULL UNIQUE COMMENT ''登录账号'',
  password_hash   VARCHAR(255) NOT NULL COMMENT ''密码加密存储'',
  real_name       VARCHAR(50) COMMENT ''真实姓名'',
  dept_id         BIGINT COMMENT ''所属部门，NULL=未分配'',
  status          TINYINT DEFAULT 1 COMMENT ''1=正常 0=禁用'',
  last_login_at   DATETIME COMMENT ''最后登录时间'',
  created_at      DATETIME,
  updated_at      DATETIME,
  INDEX idx_dept (dept_id)
) COMMENT=''用户表'';
```

## 二、通用响应格式

所有接口统一返回格式。

### 2.1 成功响应

```json
{
  "code": 200,
  "msg": "ok",
  "data": {}
}
```

### 2.2 分页响应

```json
{
  "code": 200,
  "msg": "ok",
  "data": {
    "records": [],
    "total": 100,
    "page": 1,
    "size": 20
  }
}
```

### 2.3 错误响应

```json
{
  "code": 400,
  "msg": "参数错误",
  "data": null
}
```

> 说明：code 非 200 即为失败，msg 为错误提示文案。


## 四、部门管理

### 4.1 页面路径

`/system/department`

### 4.2 页面布局

```
┌──────────────────────────────────────────────────────────┐
│ PageHeader: 部门管理                                        │
├──────────────────────┬─────────────────────────────────┤
│ 部门树（约30%）          │ 用户列表（约70%）                      │
│                       │                                   │
│ ├─[企业名]             │ 姓名 | 账号 | 角色 | 状态 | 操作       │
│ │ ├─财务部            │                                   │
│ │ └─技术部            │                        [+ 新增用户]  │
│ └─未分配              │                                   │
│                       │                                   │
│ [+ 新增部门]             │                                   │
│ [编辑部门] [删除部门]    │                                   │
└──────────────────────┴─────────────────────────────────┘
```

- 左侧选中部门 → 右侧显示该部门用户列表
- 右侧新增用户 → 自动归属当前选中部门
- 根节点名称：来自 application.yml 的 system.enterprise-name，后续迁移到系统配置
- 未分配用户：dept_id=NULL 的用户，登录后提示「等待分配」，不进入系统

### 4.3 API 接口

```
GET    /system/departments/tree          部门树（含根节点，含状态）
POST   /system/departments               新增部门
PUT    /system/departments/{id}          编辑部门（名称、排序、状态）
DELETE /system/departments/{id}          删除部门（含级联）
GET    /system/departments/{id}/users    获取部门下用户列表（分页）
```

### 4.4 删除部门级联逻辑

```
DELETE /departments/{id}
  1. 查询子部门 → 递归删除所有子部门
  2. 查询本部门用户 → dept_id 置 NULL（归入未分配）
  3. 删除本部门记录
```

### 4.5 禁用部门级联逻辑

登录验证时增加部门状态检查：

```
user = findUserByUsername(username)
dept = findDeptById(user.dept_id)
if (dept.status == 0) {
  return { code: 401, msg: "所属部门已停用，请联系管理员" }
}
```

### 4.6 删除确认对话框

删除前弹窗提示影响范围：

```
标题：确认删除部门
内容：
  - 将同时删除 N 个子部门
  - M 名用户将转为未分配状态
  - 此操作不可恢复

[取消]  [确认删除]
```

## 五、用户管理

### 5.1 页面路径

`/system/user`

### 5.2 页面布局

```
┌────────────────────────────────────────────────────────────┐
│ PageHeader: 用户管理                            [+ 新增用户] │
├────────────────────────────────────────────────────────────┤
│ 部门：<全部>  状态：<全部>  账号：________ [搜索]             │
├────────────────────────────────────────────────────────────┤
│ 姓名 | 账号 | 所属部门 | 角色 | 状态 | 操作                   │
│ 张三 | zhangsan | 财务部 | 财务录入 | 正常 | [编辑][禁用][删除] │
└────────────────────────────────────────────────────────────┘
```

### 5.3 API 接口

```
GET    /system/users                       用户列表（分页+筛选）
POST   /system/users                       新增用户
PUT    /system/users/{id}                  编辑用户（姓名、账号、部门）
DELETE /system/users/{id}                  删除用户
PUT    /system/users/{id}/password         重置密码（生成随机密码或固定初始密码）
PUT    /system/users/{id}/status            启用/禁用用户
PUT    /system/users/{id}/dept              调整用户所属部门
```

### 5.4 新增/编辑用户

**新增字段：**
- 账号（username）：必填，唯一
- 姓名（real_name）：必填
- 初始密码（password）：必填
- 所属部门（dept_id）：必填（下拉选择）
- 状态（status）：默认正常

**编辑字段：**
- 姓名、所属部门、状态（账号不可改）

### 5.5 禁用用户

禁用后该用户立即无法登录，不影响当前会话（JWT 无状态）。

## 六、初始化数据

### 6.1 根部门

系统首次启动时自动插入一条根部门：

```sql
INSERT INTO org_department (id, name, parent_id, sort_order, status)
VALUES (1, '我的公司', NULL, 0, 1);
```

> 企业名称后期从系统配置读取。

### 6.2 初始管理员

系统首次启动时可选择是否创建初始 admin 账号（dept_id=NULL），或在启动脚本/文档中说明手动创建方式。
